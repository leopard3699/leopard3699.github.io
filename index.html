tuxedo多域网关及跨域路由双活问题

发表于 2016-03-29

Tuxedo多域及跨域路由双活

由目前的单域网关转为多域网关

技术实现：

配置tuxedo多域网关组，根据UBB中的DDR路由规则，可以指定某一组跨域服务走域网关A,其他跨域服务走域关B，来缓解目前单域网关的压力.

配置图示





技术验证

ubb中指定routing方式，将对871101的跨域调用走域网关GW1，将对871104的跨域调用走域网关GW2，DM中指定远程域routing方式，指定GW1下的871101连接ubps50，GW2下的871104连接ubps70.分别调用871101及871104，测试发现GW1及GW2网关调用次数增加1且远程域50、70均找到调用记录

跨域路由如何解决双活

技术实现

修改tuxedo DM_ROUTING中路由表达式的RANGES属性

配置图示



技术验证

在ubps50及ubps70均正常时，跨域路由访问ubps50，当ubps50挂掉时，tuxedo自动将对4的跨域调用，路由到ubps70上.

8版后台环境搭建(应用及数据库)

发表于 2016-03-24

搭建后台环境

应用服务安装及配置

Server打包

1.在其他环境找一份完备的Server目录，打包下载到待安装环境
2.Server文件夹说明 

bin      ——后台服务的可执行文件        
lib      ——后台服务的动态链接文件目录      
move     ——保单转移数据目录      
etc         ——后台服务的配置目录，包括应用服务环境和服务配置文件。  
dev      ——存放TLOG文件是各个site1上的事务日志文件。  
xml_data ——存放保全，理赔等打印产生XML信息  
xml_data/excel ——日结清单打印产生XML信息   
trans    ——存放银行转帐的数据文件   
log      ——存放交易日志及tuxedo日志及分红数据文件

环境变量配置

1.修改$HOME/.bash_profile文件,在.bash_profile最后增加一行. 

. ./Server/etc/setenv.sh

2.修改$HOME/Server/etc/setenv.sh中的环境变量设置,应用服务需要修改设置有 

WSNADDR：UBBCONFIG中WSL的地址
CBPSDIR：后台服务安装目录

3.修改后好setevn.sh文件,重新登录系统

UBB、DM配置

tmunloadcf >ubb

*MACHINES
    "k1rhel582324"    LMID="CBPS24"  #修改此处的k1rhel582324、CBPS24改为新环境名字
    TUXCONFIG="/datafs/fs/Server/etc/tuxconfig"  
    TUXDIR="/bea/tuxedo111120/tuxedo11gR1" 
    APPDIR="/datafs/fs/Server/bin" 
    TLOGDEVICE="/datafs/fs/Server/dev/TLOG" 
    TLOGNAME="TLOG"
    TLOGSIZE=500
    ULOGPFX="/datafs/fs/Server/log/ULOG" 

*GROUPS

*RMS

*NETGROUPS

*SERVERS
    "WSL"    SRVGRP="TUXSYSTEM"    SRVID=9999
    CLOPT="-A -t --  -n //10.31.23.24:18300 -m 3 -M 10 -x 1 #修改WSL的IP地址配置         -p    21701    -P    21799"

rm $CBPSDIR/etc/tuxconfig

4.导入ubb 

tmloadcf  ubb

4.导入DM 

dmloadcf dm

数据库设置

删除原数据库

一种直接删除数据库，另外一种只删除数据库实例上的用户及数据文件

删除数据库实例

sqlplus / as sysdba
SQL> shutdown immediate;
SQL> startup restrict mount;
SQL> drop database cbps;
Database dropped
SQL> exit

删除原用户

drop user biz_proc cascade;
drop user cl_biz1 cascade;
drop user cl_biz2 cascade;
drop user cl_cntap cascade;
drop user cl_cntr1 cascade;
drop user cl_cntr2 cascade;
drop user cl_interface cascade;
drop user cl_cust1 cascade;
drop user code cascade;
drop user product cascade;
drop user incrementcbpsv8 cascade;
drop user un_biz cascade;

删除原表空间

drop tablespace xxx including contents;
删除原表空间数据文件
rm /datafs/oradata/cbps/v8data*

创建新库

创建数据库实例

在未创建数据库实例时执行如下语句

dbca -silent -createDatabase -gdbname cbps -sid cbps -sysPassword oracle -systemPassword oracle -datafileDestination /datafs/oradata -recoveryAreaDestination /datafs/oradata -storageType FS -databaseType OLTP -characterSet ZHS32GB18030 -nationalCharacterSet AL16UTF16 -templateName $ORACLE_HOME/assistants/dbca/templates/General_Purpose.dbc

计算原表空间大小

在导出库执行如下SQL

select
b.file_name 物理文件名,
b.tablespace_name 表空间,
b.bytes/1024/1024 大小,
(b.bytes-sum(nvl(a.bytes,0)))/1024/1024  已使用,
substr((b.bytes-sum(nvl(a.bytes,0)))/(b.bytes)*100,1,5)  利用率
from dba_free_space a,dba_data_files b
where a.file_id=b.file_id
group by b.tablespace_name,b.file_name,b.bytes
order by b.tablespace_name;

创建表空间

根据上面计算出的表空间大小，按需创建表空间

create tablespace tjv8dttbs datafile '/datafs/oradata/cbps/v8data01.dbf' size 32000M autoextend off;
alter tablespace tjv8dttbs add datafile '/datafs/oradata/cbps/v8data02.dbf' size 32000M autoextend off;
alter tablespace tjv8dttbs add datafile '/datafs/oradata/cbps/v8data03.dbf' size 32000M autoextend off;
alter tablespace tjv8dttbs add datafile '/datafs/oradata/cbps/v8data04.dbf' size 32000M autoextend off;
alter tablespace tjv8dttbs add datafile '/datafs/oradata/cbps/v8data05.dbf' size 32000M autoextend off;
alter tablespace tjv8dttbs add datafile '/datafs/oradata/cbps/v8data06.dbf' size 32000M autoextend off;

alter tablespace tjv8dttbs add datafile '/datafs/oradata/cbps/v8data08.dbf' size 32000M autoextend off;

创建DBLINK

sys用户登录数据库，创建数据库链接，通过DBLINK方式导库（也可通过常规的EXPDP、IMPDP操作）

sqlplus sys/oracle@cbps    as sysdba

create database link link101 connect to cl_biz1 identified by wasd4321 using '(DESCRIPTION =     (ADDRESS_LIST =       (ADDRESS = (PROTOCOL = TCP)(HOST = 10.31.17.101)(PORT = 1521))     )     (CONNECT_DATA =       (SERVICE_NAME = cbps)     )   )';

导入整库

注意源库中的表空间，需将所涉及的所有表空间都导入到新库，耗时较长，按目前天津库情况一般需要8-12小时

nohup impdp system/oracle@cbps network_link=link101 logfile=impdp101_1001.log schemas=BIZ_PROC,CL_BIZ1,CL_BIZ2,CL_CNTAP,CL_CNTR1,CL_CNTR2,CL_INTERFACE,CL_CUST1,CODE,PRODUCT,UN_BIZ,incrementcbpsv8 remap_tablespace=TJV8DTTBS:tjv8dttbs,TJV8IDXTBS:tjv8dttbs  &

导入public同义词

有时候有些pulic的同义词未导入到新库，需要重新创建这些同义词
首先先在源库查询pulic的同义词，将SQL查询结果在sqlplus下执行

SELECT 'CREATE  PUBLIC SYNONYM '||synonym_name||' FOR '||table_owner||'.'||table_name||';' FROM DBA_SYNONYMS WHERE 
table_owner in ('BIZ_PROC','CL_BIZ1','CL_BIZ2','CL_CNTAP','CL_CNTR1','CL_CNTR2','CL_INTERFACE','CL_CUST1','CODE','PRODUCT','UN_BIZ')  
AND owner='PUBLIC'

数据库调整

调整数据库链接数

修改数据库链接数为1500，并重启数据库

sqlplus sys/oracle@cbps as sysdba
SQL> alter system set processes=1500 scope=spfile;
SQL> shutdown immediate
SQL> startup

调整用户过期设置

sqlplus sys/oracle@cbps as sysdba
SQL>ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;

重新编译失效对象

将如下SQL的执行结果，在sqlplus中执行

SELECT 'ALTER ' || object_type || ' ' || owner || '.' || object_name ||
   ' COMPILE;'
FROM dba_objects
WHERE status <> 'VALID';
或直接编译schema
execute utl_recomp.recomp_serial('SCOTT');

重新对数据库进行统计分析

导入过程中会丢失数据库统计分析信息，导致执行计划错误，数据库性能较差，可执行如下SQL重新进行统计分析

Exec dbms_stats.gather_schema_stats(ownname=>'BIZ_PROC',cascade=> TRUE, degree =>16);
Exec dbms_stats.gather_schema_stats(ownname=>'CL_BIZ1',cascade=> TRUE, degree =>16);
